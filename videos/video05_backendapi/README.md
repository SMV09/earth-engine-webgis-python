Build a WebGIS Backend API with Flask & Google Earth Engine (NDVI Tutorial) -Phase 2

Explore WebGIS Architecture to see how the browser, the backend, and the cloud work together as a single system. We then move into Choosing a Framework, where I explain why we use Flask instead of other tools like Django or Streamlit. You will learn about the Data Flow, seeing exactly how a date you pick on a website travel to the GEE and returns as an NDVI result. We will also cover Project Structure as two clear files: config.py for our settings and app.py for the server. Finally, run and test the API endpoint by providing URL for date range to display NDVI result in JSON format. By the end of this video, you will have a live backend service ready for WebGIS!
